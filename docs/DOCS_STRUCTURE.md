## 为什么这样划分？

**公开文档**：帮助其他开发者理解和使用项目
**内部文档**：记录项目演进过程、设计决策、内部规范，属于项目知识库

这样既保持了开源项目的简洁性，又保留了完整的项目历史和上下文。
